## Работа с тегами (git tag)

Команда `git tag` используется для создания, просмотра и удаления тегов в Git. Теги часто применяются для маркировки важных точек в истории проекта, например, для релизов или значимых версий. Теги могут быть аннотированными (с метаданными) или простыми (только хешом коммита).

### Синтаксис:
```bash
git tag [options] [tag] [commit]
```
- `tag` — имя тега, который вы хотите создать.
- `commit` — необязательный параметр, который указывает коммит, с которым будет связан тег. Если не указать коммит, тег будет привязан к текущему состоянию репозитория.

### Примеры использования:
1. Создание простого тега:

```bash
git tag v1.0
```
Эта команда создаёт простой тег с именем `v1.0`, который будет связан с текущим коммитом.

2. Создание аннотированного тега: Аннотированные теги содержат дополнительную информацию, такую как имя автора, дата и сообщение.

```bash
git tag -a v1.0 -m "Первый релиз проекта"
```
Здесь флаг -a указывает на создание аннотированного тега, а -m задаёт сообщение, связанное с тегом.

3. Создание тега для конкретного коммита: Чтобы создать тег для определённого коммита, укажите его хеш:

```bash
git tag v1.0 9fceb02
```
В этом примере тег `v1.0` будет создан для коммита с хешем `9fceb02`.

4. Просмотр всех тегов: Для просмотра всех тегов в репозитории используйте команду:

```bash
git tag
```
Эта команда выводит список всех тегов.

5. Просмотр информации о конкретном аннотированном теге: Чтобы увидеть информацию о теге (например, сообщение тега), используйте команду:

```bash
git show v1.0
```
6. Удаление тега: Чтобы удалить тег, используйте команду:

```bash
git tag -d v1.0
```
Эта команда удаляет локальный тег `v1.0`. Тег удаляется только локально, а не в удалённом репозитории.

7. Отправка тегов в удалённый репозиторий: Для отправки тегов на сервер используйте команду:

```bash
git push origin v1.0
```
Это отправит тег `v1.0` в удалённый репозиторий. Если нужно отправить все теги, используйте:

```bash
git push --tags
```
8. Удаление тега из удалённого репозитория: Для удаления тега из удалённого репозитория используйте команду:

```bash
git push --delete origin v1.0
```
Это удаляет тег `v1.0` из удалённого репозитория.

### Полезные опции:
- `-a` — создаёт аннотированный тег, который сохраняет информацию о тегере (имя, дата и сообщение).

```bash
git tag -a v1.0 -m "Message"
```
- `-d` — удаляет тег.

```bash
git tag -d v1.0
```
- `--list` — отображает список тегов, возможно с фильтрацией.

```bash
git tag --list "v*"
```
Это покажет все теги, начинающиеся с `v`.

- `-v` — отображает информацию об аннотированном тегу.

```bash
git tag -v v1.0
```
- `--contains <commit>` — показывает все теги, которые содержат указанный коммит.

```bash
git tag --contains abc123
```

### Когда использовать git tag?
1. Когда необходимо создать метку для важной точки в истории проекта, например, для релиза.
2. Для маркировки конкретных версий или стабильных состояний кода.
3. Для передачи тегов в удалённые репозитории, чтобы другие пользователи могли использовать их для конкретных версий проекта.
