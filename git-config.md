## Настройка Git `git config`

Команда `git config` используется для настройки параметров и конфигурации Git на уровне системы, репозитория или пользователя. Эти параметры могут включать имя пользователя, адрес электронной почты, редактор по умолчанию, настройки поведения Git и другие предпочтения. Git позволяет вам настроить конфигурацию на разных уровнях, в зависимости от ваших нужд.

### Синтаксис:
```bash
git config [options] <parametrs> [value]
```
- `<parametrs>`— это конфигурационный параметр, который вы хотите изменить или получить (например, имя пользователя или редактор).
- `[value]` — это новое значение для конфигурационного параметра.

### Уровни конфигурации:
1. **Система** (`--system`) — применяются ко всем пользователям на системе. Для изменения параметров на этом уровне требуются права администратора.
2. **Пользователь** (`--global`) — применяются ко всем репозиториям пользователя. Это уровень по умолчанию, если не указать другой.
3. **Репозиторий** (`--local`) — применяются только к текущему репозиторию. Это уровень по умолчанию, если не указан другой уровень.

### Примеры использования:
1. Установка имени пользователя и электронной почты (для коммитов): Чтобы настроить имя пользователя и адрес электронной почты, которые будут использоваться для коммитов, выполните следующие команды:

```bash
git config --global user.name "John Doe"
git config --global user.email "john.doe@example.com"
```
Эти параметры будут использоваться для всех репозиториев пользователя. Если нужно настроить их только для конкретного репозитория, уберите флаг `--global`.

2. Просмотр текущих настроек: Чтобы просмотреть все текущие настройки, используйте команду:

```bash
git config --list
```
Это выведет список всех настроек, применённых в текущем репозитории.

3. Установка редактора для Git: Чтобы задать редактор по умолчанию для Git, используйте команду:

```bash
git config --global core.editor "nano"
```
В данном случае редактор будет установлен на `nano`. Вы можете использовать любой другой редактор, например, `vim` или `code`(Visual Studio Code).

4. Настройка слияния и диффов с использованием внешних инструментов: Вы можете настроить сторонние инструменты для слияния или просмотра различий (например, `meld`, `kdiff3`):

```bash
git config --global merge.tool meld
git config --global diff.tool meld
```
5. Настройка цвета в выводе Git: Для улучшения восприятия можно включить цветное оформление в выводе Git:

```bash
git config --global color.ui auto
```
Эта настройка включает цвета в выводе команд, таких как `git status` или `git diff`.

6. Просмотр конфигурации для конкретного параметра: Чтобы просмотреть значение конкретного параметра, используйте:

```bash
git config user.name
```
Эта команда выведет имя пользователя, которое настроено в текущем репозитории (или на глобальном уровне, если оно не задано для репозитория).

7. Изменение конфигурации на уровне репозитория: Если вы хотите настроить параметры только для текущего репозитория, используйте флаг `--local`:

```bash
git config --local user.email "repo@example.com"
```
Эта настройка будет применяться только в текущем репозитории, а не глобально.

8. Удаление конфигурации: Чтобы удалить параметр конфигурации, используйте команду `--unset`:

```bash
git config --global --unset user.name
```
### Полезные опции:
- `--global` — применяет настройки ко всем репозиториям пользователя (на уровне пользователя).

```bash
git config --global user.name "Your Name"
```
- `--system` — применяется ко всем пользователям на системе (требуются права администратора).

```bash
git config --system core.editor vim
```
- `--local` — применяется только к текущему репозиторию.

```bash
git config --local user.name "Repo User"
```
- `--unset`— удаляет параметр конфигурации.

```bash
git config --global --unset user.email
```
- `--edit`— открывает файл конфигурации в редакторе для ручного редактирования.

```bash
git config --global --edit
```

### Когда использовать `git config`?
1. Настройка личных данных: Чтобы настроить ваше имя и email для всех коммитов, используйте `git config --global`.
2. Настройка редактора: Вы можете настроить редактор, который будет использоваться для редактирования сообщений коммитов и других файлов.
3. Конфигурация внешних инструментов: Для настройки внешних инструментов слияния и диффов, таких как `meld`, `kdiff3`, `vimdiff`, и других.
4. Персонализация выводов: Вы можете изменить вывод команд Git, например, добавить цветное оформление для улучшения восприятия.
