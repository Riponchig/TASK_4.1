## Сравнение изменений `git diff`

Команда `git diff` используется для отображения различий между различными версиями файлов в репозитории. Это помогает понять, какие изменения были внесены в файлы, с момента последнего коммита или между двумя коммитами. Команда полезна для просмотра изменений до того, как они будут зафиксированы или для анализа различий между ветками или репозиториями.

### Синтаксис:
```bash
git diff [опции] [файлы]
```

- `файлы` — список файлов, для которых вы хотите увидеть различия. Если не указаны файлы, Git покажет различия для всех изменённых файлов.

### Примеры использования:
1. Просмотр изменений между рабочей директорией и последним коммитом:

```bash
git diff
```
Эта команда покажет изменения, которые ещё не были добавлены в индекс (с помощью `git add`).

2. Просмотр изменений, добавленных в индекс, но ещё не зафиксированных:

```bash
git diff --cached
```
Команда покажет изменения, которые были добавлены в индекс с помощью `git add`, но ещё не зафиксированы с помощью `git commit`.

3. Просмотр различий между двумя коммитами:

```bash
git diff <commit1> <commit2>
```
Эта команда покажет различия между двумя указанными коммитами. Например:

```bash
git diff a1b2c3d4 e5f6g7h8
```
4. Просмотр различий между двумя ветками:

```bash
git diff branch1..branch2
```
Эта команда покажет различия между ветками `branch1` и `branch2`.

5. Просмотр изменений для конкретного файла:

```bash
git diff <файл>
```
Эта команда покажет изменения, которые были сделаны только в указанном файле.

6. Просмотр различий между рабочей директорией и конкретным коммитом:

```bash 
git diff <commit> <файл>
```
Эта команда покажет различия между рабочей директорией и состоянием файла в указанном коммите.

### Полезные опции:
- `--name-only` — показывает только имена изменённых файлов, без отображения самих изменений.

```bash
git diff --name-only
```
- `--color` — включает цветовое выделение различий для лучшего восприятия.

```bash
git diff --color
```
- `--stat` — показывает статистику изменений (количество добавленных и удалённых строк) вместо самих различий.

```bash
git diff --stat
```
- `--unified=<n>` — изменяет количество строк контекста, которые отображаются вокруг различий. По умолчанию отображается 3 строки контекста, но вы можете изменить это значение.

```bash
git diff --unified=5
```
- `--word-diff` — показывает различия на уровне слов, а не строк.

```bash
git diff --word-diff
```
