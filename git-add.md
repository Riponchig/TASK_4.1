## Добавление изменений `git add`

Команда `git add` используется для добавления изменений в индекс (staging area). Индекс — это область, где собираются изменения перед фиксацией с помощью команды `git commit`.


### Синтаксис:
```bash
git add [опции] <файл/путь>
```
- `<файл/путь>` — имя файла или путь к каталогу, который нужно добавить в индекс.
- Если указан только символ `.` (точка), добавляются все изменения в текущей директории.

### Примеры использования:
1. Добавление одного файла в индекс:

```bash
git add example.txt
```
Добавляет файл `example.txt` в индекс.

2. Добавление всех файлов в текущей директории:

```bash
git add .
```
Добавляет в индекс все изменения в текущей директории, включая новые и изменённые файлы.

3. Добавление определённой директории:

```bash
git add src/
```
Добавляет все файлы из директории `src` в индекс.

4. Добавление всех файлов, но без удаления удалённых файлов:

```bash
git add --all
```
Добавляет все файлы и изменения, включая удаление файлов, в индекс.

### Полезные опции:
1. `-p` или `--patch`

Позволяет добавлять изменения частями.

```bash
git add -p
```
2. `--dry-run`

Показывает, что будет добавлено, без реального добавления.

```bash
git add --dry-run
```
3. `-u`

Добавляет только изменённые и удалённые файлы, но не новые.

```bash
git add -u
```
### Когда использовать `git add`?
1. После внесения изменений в файлы, которые нужно зафиксировать.
2. Для добавления новых файлов в контроль версий.
3. Перед выполнением команды `git commit`.

### Примечание:
Файлы, перечисленные в `.gitignore`, не добавляются в индекс.
