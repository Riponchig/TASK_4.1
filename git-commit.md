## Фиксация изменений `git commit`

Команда `git commit` используется для фиксации изменений, добавленных в индекс (staging area). Коммит сохраняет текущие изменения в локальной истории репозитория, с привязкой к сообщению, которое описывает суть изменений.

### Синтаксис:
```bash
git commit [опции] -m "Сообщение коммита"
```
- `-m "Сообщение коммита"` — обязательный параметр, который задаёт описание для коммита.

### Примеры использования:
1. Простой коммит с сообщением:
```bash
git commit -m "Добавил новый функционал для авторизации"
```
Коммитит все изменения, добавленные в индекс, с сообщением о том, что был добавлен новый функционал.

2. Коммит после добавления всех изменений:
```bash
git add .
git commit -m "Обновление README и добавление нового файла"
```
Здесь выполняется сначала добавление всех изменений в индекс командой `git add .`, а затем эти изменения фиксируются командой `git commit`.

3. Коммит с автосохранением сообщений: Если не использовать параметр `-m`, Git откроет редактор для ввода сообщения коммита:
```bash
git commit
```
Это полезно для добавления более длинных и детализированных сообщений.

4. Коммит с несколькими изменениями:
```bash
git commit -am "Исправлены ошибки в коде и документации"
```
Параметр `-a` автоматически добавляет все изменённые файлы (кроме новых) в индекс перед коммитом.

### Полезные опции:
- `--amend` — позволяет изменить последний коммит (например, если нужно исправить сообщение или добавить пропущенные файлы).

```bash
git commit --amend
```
Это откроет редактор для изменения предыдущего сообщения коммита. Если хотите добавить файлы, которые забыли добавить в последний коммит, используйте `git add` перед этим.

- `--no-edit` — используется для изменения последнего коммита без изменения сообщения.

```bash
git commit --amend --no-edit
```
- `--date` — позволяет задать дату коммита вручную.

```bash
git commit --date="2025-01-01 12:00:00" -m "Коммит с заданной датой"
```
### Когда использовать `git commit`?

- После добавления файлов в индекс с помощью команды `git add`, чтобы зафиксировать изменения в локальной истории.
- Когда необходимо сохранить текущие изменения и перейти к другой задаче.
- Чтобы создать точку восстановления перед изменением кода.
