## Инициализация репозитория

Команда `git init` используется для создания нового локального репозитория Git в указанной директории. Она инициализирует систему контроля версий, позволяя Git отслеживать изменения в файлах и управлять историей проекта.

### Синтаксис:
```bash
git init [путь]
```
- путь (опционально) — путь к директории, где будет создан репозиторий. Если путь не указан, репозиторий создаётся в текущей директории.

### Что происходит после выполнения команды:
1. В указанной директории создаётся скрытая папка `.git`.
2. Эта папка содержит все метаданные, необходимые для работы Git:
    - Историю коммитов.
    - Настройки репозитория.
    - Индекс (staging area).

### Примеры:
1. Инициализация репозитория в текущей директории:

```bash
git init
```
После выполнения этой команды текущая директория становится Git-репозиторием.

2. Инициализация репозитория в другой директории:

```bash
git init /[путь к папке] 
```

В указанной папке создаётся репозиторий.

### Что делать после git init?
1. Добавить файлы в репозиторий:
```bash
git add .
```
2. Сделать первый коммит:
```bash
git commit -m "Первый коммит"
```
