## Просмотр истории коммитов `git log`

Команда `git log` используется для просмотра истории коммитов в репозитории. Она позволяет увидеть, какие коммиты были сделаны, кем и когда, а также какие изменения были внесены в код. Это полезно для анализа истории проекта, отслеживания изменений и поиска информации о коммитах.

### Синтаксис:
```bash
git log [options]
```
- `options` — различные параметры для настройки вывода, такие как формат коммитов, количество выводимых коммитов и другие.

### Примеры использования:
1. Просмотр всей истории коммитов:

```bash
git log
```
Эта команда выводит список всех коммитов в репозитории, начиная с самого последнего. Для каждого коммита будет показан хеш, автор, дата и сообщение коммита.

**Пример вывода:**

```sql
Копировать
Редактировать
commit abc1235
Author: John Doe <john.doe@example.com>
Date:   Mon Jan 10 12:00:00 2025 +0300

    Fix bug in authentication module

commit abc1234
Author: John Doe <john.doe@example.com>
Date:   Sun Jan 10 14:30:00 2025 +0300

    Add new feature to dashboard
```
2. Просмотр истории с ограничением на количество коммитов:

```bash
git log -n 5
```
Покажет последние 5 коммитов.

3. Просмотр истории в компактном формате (один коммит в строке):

```bash
git log --oneline
```
Это позволяет вывести историю коммитов в сокращённом виде, где каждый коммит отображается как одна строка, состоящая из короткого хеша и сообщения коммита.

**Пример вывода:**

```sql
abc1234 Fix bug in authentication module
abc1235 Add new feature to dashboard
```
4. Просмотр истории с выводом диффов:

```bash
git log -p
```
Покажет историю коммитов с диффами (различиями), т.е. изменениями в файлах, которые были сделаны в каждом коммите.

5. Просмотр коммитов для конкретного файла:

```bash
git log <file>
```
Покажет историю коммитов только для указанного файла, что полезно для отслеживания изменений конкретного файла в проекте.

6. Просмотр истории с указанием периода времени:

```bash
git log --since="2025-01-01" --until="2025-01-18"
```
Покажет коммиты, сделанные в период с 1 января 2025 года по 18 января 2025 года.

7. Просмотр истории с фильтрацией по автору:

```bash
git log --author="John Doe"
```
Покажет только те коммиты, которые были сделаны автором с указанным именем.

8. Просмотр истории для конкретной ветки:

```bash
git log <branch>
```
Выведет историю коммитов для указанной ветки. Например:

```bash
git log feature-branch
```

### Полезные опции:
- `--graph` — отображает графическое представление истории коммитов, что полезно при анализе ветвлений и слияний.

```bash
git log --graph
``` 
- `--pretty=format:"<format>"` — позволяет настроить формат вывода. Например, вы можете вывести только хеши и сообщения коммитов:

```bash
git log --pretty=format:"%h - %s"
```
Это покажет только сокращённый хеш коммита и его сообщение.

- `--abbrev-commit` — сокращает хеш коммита до первых 7 символов.

```bash
git log --abbrev-commit
```
- `--stat`— показывает статистику изменений в файлах для каждого коммита (количество добавленных и удалённых строк).

```bash
git log --stat
```
- `--reverse` — выводит историю коммитов в обратном порядке (от старого к новому).

```bash
git log --reverse
```
### Когда использовать git log?
1. Когда нужно просмотреть историю изменений в проекте и понять, какие коммиты были сделаны.
2. Для анализа изменений в конкретных файлах или в определённый период времени.
3. Когда необходимо исследовать, кто и когда внес определённые изменения в проект.
4. Для изучения ветвлений и слияний в репозитории.
