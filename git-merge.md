## Слияние веток (git merge)

Команда `git merge` используется для слияния изменений из одной ветки в другую. Это одна из основных команд для работы с ветками в Git. Она позволяет интегрировать код из другой ветки в текущую ветку, обеспечивая синхронизацию и объединение работы нескольких участников проекта.

### Синтаксис:
```bash
git merge [branch]
```
- `branch` — это имя ветки, изменения из которой вы хотите слить с вашей текущей веткой.

### Примеры использования:
1. Слияние ветки `feature-branch` в текущую ветку: Для того чтобы объединить изменения из ветки `feature-branch` с текущей веткой, используйте:

```bash
git merge feature-branch
```
Это будет сливать изменения из `feature-branch` в ветку, на которой вы находитесь в данный момент.

2. Слияние ветки с помощью опции `--no-ff`: Использование опции `--no-ff` заставляет Git всегда создавать новый коммит слияния, даже если это можно было бы выполнить с помощью "быстрого вперёд" (fast-forward).

```bash
git merge --no-ff feature-branch
```
3. Слияние ветки с опцией `--squash`: Опция `--squash` позволяет объединить все изменения из сливаемой ветки в один коммит. Это полезно, если вы хотите, чтобы история оставалась чистой и не загромождалась множеством мелких коммитов.

```bash
git merge --squash feature-branch
git commit -m "Объединённые изменения из feature-branch"
```
В этом случае слияние не будет завершено автоматически, и вам нужно будет выполнить коммит вручную.

4. Слияние ветки с разрешением конфликтов: Если при слиянии возникают конфликты, Git уведомит вас о том, какие файлы требуют разрешения. После разрешения конфликтов нужно будет зафиксировать изменения с помощью нового коммита:

```bash
git merge feature-branch
git add <исправленные файлы>
git commit
```
### Полезные опции:
- `--ff-only`— позволяет выполнять слияние только в случае "быстрого вперёд" (fast-forward). Если слияние невозможно, например, если на целевой ветке были сделаны изменения, команда завершится с ошибкой.

```bash
git merge --ff-only feature-branch
```
- --no-ff — предотвращает слияние с использованием "быстрого вперёд" и всегда создаёт коммит слияния. Это полезно, если вы хотите, чтобы история сохраняла коммиты из сливаемой ветки, даже если можно было бы обойтись без коммита слияния.

```bash
git merge --no-ff feature-branch
```
- `--squash` — сливает изменения из ветки, но не создает коммит слияния. Вместо этого все изменения аккумулируются в одном коммите.

```bash
git merge --squash feature-branch
```

### Когда использовать git merge?
1. Когда вы хотите объединить работу из разных веток, например, после завершения работы над новой фичей или исправлением ошибок.
2. При интеграции изменений из ветки, созданной для другой задачи, в основную ветку, например, `main` или `master`.
3. Когда нужно объединить изменения из удалённой ветки с вашей локальной веткой (например, после `git pull`).
